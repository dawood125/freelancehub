# Agent Workspace

Last updated: 2026-04-15

Purpose:
- Keep long-term project context in one place so planning continuity is never lost.
- Track what was done, why it was done, and what comes next.
- Keep implementation quality guardrails for security, edge cases, scalability, architecture, performance, and coding standards.

How to use this folder:
1. Read `00-core/project-snapshot.md` before starting a new implementation session.
2. Follow `90-roadmap/current-plan.md` for the active milestone order.
3. Log each important decision in `00-core/decision-log.md`.
4. Update the domain checklists before closing a sprint.

Folder map:
- `00-core/`: project snapshot, active agreement, decision log
- `10-architecture/`: current architecture and module boundaries
- `20-security/`: security priorities and implementation checklist
- `30-edge-cases/`: critical edge cases by domain
- `40-scalability/`: scaling notes and guardrails
- `50-performance/`: frontend/backend performance actions
- `60-coding-style/`: coding conventions and refactor rules
- `70-skills/`: learning/mentor workflow and delivery format
- `80-testing/`: test strategy and minimum release checks
- `90-roadmap/`: current implementation plan and next milestones
